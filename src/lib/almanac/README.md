## 需外接：官方天文資料源（`AstronomicalProvider`）

農曆（定朔／中氣／閏月）、節氣（定氣）、年柱（立春分年）、月支（節分月）需中央氣象署天文資料。
實作 `AstronomicalProvider` 介面（見 `index.ts`）並傳入 `computeDayRecord(y,m,d,{astro})` 即可點亮。
未接前 `status.astronomicalDataConnected=false`，農曆／節氣／年月柱／宜忌不顯示。

## 考據化原則（C.6 — 本模組差異化核心）

進階層**不只給結論，必附出處與推導**：每條宜忌／值神／神煞結論帶 `derivation`（由哪些值神／神煞依何規則判定）與 `sources[]`（《欽定協紀辨方書》為基準，包含但不限於）。諸說不一時並陳，不裁定。

> 這是與市面農民曆 app 的分水嶺：別人只給「今日宜嫁娶」，本站給「依《協紀辨方書》某神煞，此日宜嫁娶」。

## 基準宣告（C.8）

- 日界：民用日 00:00 換日；命理子初換日之日柱另標基準，不混用。
- 時區：農曆／節氣／定朔以東經 120°（UTC+8）為準。
- 閏月聖誕：閏月不另慶誕，歸該月正月；他俗諸說並陳。
- 有效年限：依所採官方天文資料涵蓋年限；超範圍回「超出曆算範圍」，不外推硬算。
